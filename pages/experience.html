Discord MongoDB Bot

Using Discord's Python API and Mongo Database, I created a bot that can be used in the tabletop role playing game Dungeons and Dragons. To use the bot, a user would upload the stat sheet for the character that they want to use for the game, and the bot will automatically parse the stat sheet for its information then upload said information to MongoDB. Once data has been uploaded, the bot can now perform any roll required using the uploaded stat sheet. For example, if a user is performing a strength check or an attack roll, the bot would automatically roll the required dice, add whatever bonuses the user's character has, and then display the randomly rolled results in the Discord chat. Users can also change the color of the bot, print out their stat sheet in the chat.

Selenium Repost Bot

Using Selenium webdriver and a bit of Python, I created a bot for a local cat breeding business to increase their online presence. I was the lead designer and programmer for this project. The cat breeders frequently used a site called gokitty.com, where cat owners list their cats up for adoption to potential customers. To save the business time, my bot was commissioned. My bot essentially took information from whatever original posts the business made, copied all its data, and then made fresh posts with the same information and pictures. This enabled the business to frequently shoot to the top of the results page because the site displayed new posts first, so the business would display its inventory to more potential buyers.

Dynamic Programming Practice

In my second Data Structures & Algorithms class, I completed a dynamic programming assignment using Java. In this assignment, I was given a list of blocks with respective integer values. My task was to solve the following question: if I can only take a block that is not next to one that has already been taken, what is the maximum possible sum that can be attained from the set? To avoid extensive an extensive runtime that a recursive solution could cause and to avoid repeated calculation, I needed to use dynamic programming to save space and time to get the optimal solution to the problem.
